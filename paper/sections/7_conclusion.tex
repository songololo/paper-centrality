\section{Discussion and Conclusions}\label{discussion}

The empirical results for Madrid are broadly consistent with the theoretical hypothesis. Regarding \textbf{RQ1}, we observe a clear distinction in behaviour: mathematically defined \emph{Closeness} exhibits sign reversal when applied to localised street network analysis, showing negative associations with land-use accessibility where the anticipated direction is positive. \emph{Normalised Closeness} does not remove this pattern---it yields negligible associations. By contrast, \emph{Improved Closeness} and \emph{Harmonic Closeness} yield values that increase with subgraph size and proximity. Regarding \textbf{RQ2}, these behavioural differences are detectable in the empirical associations: the same qualitative pattern appears for stand-alone Food \& Beverage and Retail accessibility (200m) and for a PCA-based summary index (PCA~1, reported as a robustness/compression check), as well as for origin-destination trip counts, for length-weighted and unweighted variants, and for both metric and geometric distances.

The NAIN formulation ($N^{1.2}$), introduced in \cite{Hillier2012}, represents an intermediate case. Relative to \emph{Normalised Closeness}, it tends to yield stronger associations under localised analysis, but it does not fully recover the behaviour of \emph{Improved Closeness}. NAIN was derived empirically with the aim of aligning values across different distance radii, not to address the underlying scaling issue with varying subgraph sizes.

Many packages implement a form of \emph{Improved Closeness}, \emph{Harmonic Closeness}, or both, though implementations vary. \texttt{depthmapX} \cite{turner_depthmapx_2020}, \texttt{Place Syntax Tool} \cite{stahle_place_2023}, \texttt{cityseer} \cite{simons_cityseer_2023}, \texttt{NetworkX} \cite{hagberg_networkx_2008}, and \texttt{igraph} \cite{csardi_igraph_2006} all implement variants in this family. Other packages take different approaches: \texttt{sDNA} \cite{cooper_sdna_2020} outputs a gravity-like measure (NQPD), while the \texttt{Urban Network Analysis} toolbox \cite{Sevtsuk2012} provides a \emph{Gravity Index}.

The reproducibility problem affects researchers who implement closeness centrality from cited formulas without access to original code, or who review or teach network centrality methods based on commonly cited formulations. We suspect that one reason this misattribution persists is that even though \emph{Normalised Closeness} is typically cited, many studies rely on packages that implement forms of closeness that produce the anticipated direction under localised analysis. In such cases, outputs may appear plausible even if users are under mistaken impressions about the formulation, while replication without code can yield different results, potentially with reversed sign or negligible magnitude.

These ambiguities motivate further investigation into how sensitive reported associations are to formulation choice and distance heuristic. Earlier studies that cite \emph{Normalised Closeness} without providing open code, that select small geographic areas, or that do not buffer boundaries to control edge effects \cite{Turner2007, Porta2006} may merit re-examination because the relationship between cited methods and computed results is unclear. We acknowledge that some studies using \emph{Normalised Closeness} may have operated under conditions where the distinction matters less---for example, global analysis on fixed graphs, or localised analysis at thresholds where subgraph sizes vary minimally.

We therefore suggest the use of openly available reference datasets and reproducible workflows, as we have made available in this study, so that different measures can be benchmarked openly and collaboratively against common points of reference. This would facilitate the evaluation of newly available methods against established variants while accounting for differences which might otherwise be attributable to variations in urban context or data representation. It would also help uncover situations such as the widespread misconceptions on closeness centralities discussed in this study.

This study uses a single city (Madrid); correlation magnitudes will differ across urban contexts depending on street network morphology, density gradients, and land-use distributions. However, our contribution concerns the existence and direction of formulation differences, not their precise magnitude. The theoretical mechanism is mathematical: \emph{Closeness} conflates subgraph size with average distance; \emph{Normalised Closeness} neutralises meaningful variation; \emph{Improved Closeness} and \emph{Harmonic Closeness} avoid this conflation through different mechanisms---the former by dividing by average distance, the latter by summing distance reciprocals. This mechanism arises in settings where subgraph sizes vary by origin, which is one reason domain-specific packages often adopt these formulations.

This has implications for interpreting existing literature: studies using \texttt{Depthmap} or similar tools may have produced similar outputs even if cited formulations differ from the computed ones; studies using generic network packages or custom implementations may benefit from checking which formula was actually computed; and comparative evaluations of closeness formulations should verify that implementations match cited formulas.

\subsection{Practical Guidance}

For localised street network analysis, we suggest:
\begin{enumerate}
  \item Prefer closeness formulations that scale with varying subgraph sizes, such as \emph{Improved Closeness}, \emph{Harmonic Closeness}, or \emph{Gravity}-based measures;
  \item Document the precise formulation used, not just the generic term `closeness';
  \item Provide code or specify the software package and version to enable replication;
  \item When using generic network packages, check that the closeness implementation is appropriate for localised analysis.
\end{enumerate}

\subsection{Limitations}

This study uses a single city (Madrid), which is appropriate because our argument is mathematical: the theoretical reasons why formulations diverge under localised analysis apply in settings where subgraph sizes vary by origin. The Madrid case illustrates that this divergence has empirically observable consequences. Correlation magnitudes will differ across urban contexts (e.g., cities with linear high streets, polycentric structures, or car-oriented development), but our contribution concerns the existence and direction of formulation differences, not their precise magnitude. The empirical analysis is diagnostic and cross-sectional: correlations reveal that formulations behave differently, not that any particular formulation is a definitive measure of urban accessibility. We do not model causal relationships between centrality and land-use.

\subsection{Conclusion}

The divergence between cited closeness formulations and software implementations creates a reproducibility barrier in street network analysis. Mathematical \emph{Closeness} exhibits sign reversal under localised analysis; \emph{Normalised Closeness} yields negligible associations; \emph{Improved} and \emph{Harmonic Closeness} yield values that increase with subgraph size and proximity. Researchers can reduce ambiguity by using formulations appropriate for localised analysis and documenting computational methods precisely. We provide open code and data to support reproducibility and to serve as a reference for future methodological comparisons.
