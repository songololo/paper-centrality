\section{Discussion and Conclusions}\label{discussion}

The empirical results for Madrid are consistent with the theoretical hypothesis. Regarding \textbf{RQ1}, mathematical \emph{Closeness} exhibits sign reversal under localised analysis, showing negative associations where positive ones are anticipated. \emph{Normalised Closeness} does not resolve this---it yields negligible associations. \emph{Improved Closeness} and \emph{Harmonic Closeness} yield values that increase with subgraph size and proximity. Regarding \textbf{RQ2}, these differences are detectable empirically: the same pattern appears for stand-alone accessibility measures, a PCA-based summary index, and origin-destination trip counts, across weighted and unweighted variants and both metric and geometric distances.

The NAIN formulation ($N^{1.2}$) \cite{Hillier2012} is an intermediate case: it yields stronger associations than \emph{Normalised Closeness} but does not fully recover the behaviour of \emph{Improved Closeness}, having been derived to align values across distance radii rather than to address subgraph scaling.

The reproducibility problem primarily affects researchers who implement closeness from cited formulas without access to original code, or who teach network centrality based on commonly cited formulations. We suspect this misattribution persists because many studies rely on packages that already implement variants accounting for varying subgraph sizes: outputs appear plausible even if users hold mistaken impressions about the formulation, while replication without code can yield different results. Most domain-specific packages implement forms of \emph{Improved} or \emph{Harmonic Closeness}, though implementations vary \cite{turner_depthmapx_2020, stahle_place_2023, simons_cityseer_2023, hagberg_networkx_2008, csardi_igraph_2006}.

Earlier studies citing \emph{Normalised Closeness} without open code, using small geographic areas, or lacking edge-effect controls \cite{Turner2007, Porta2006} may merit re-examination, though some may have operated under conditions where the distinction matters less. We suggest the use of openly available reference datasets and reproducible workflows so that different measures can be benchmarked collaboratively.

\subsection{Practical Guidance}

For localised street network analysis, we suggest:
\begin{enumerate}
  \item Use closeness formulations that account for varying subgraph sizes, such as \emph{Improved Closeness}, \emph{Harmonic Closeness}, or \emph{Gravity}-based measures;
  \item Document the precise formulation used, not just the generic term `closeness';
  \item Provide code or specify the software package and version to enable replication;
  \item When using generic network packages, check how the closeness implementation handles varying subgraph sizes under localised analysis.
\end{enumerate}

\subsection{Limitations}

This study uses a single city (Madrid), appropriate because our argument is mathematical: the reasons formulations diverge under localised analysis apply wherever subgraph sizes vary by origin. Correlation magnitudes will differ across urban contexts, but our contribution concerns the existence and direction of formulation differences, not their precise magnitude. The analysis is diagnostic and cross-sectional: correlations reveal that formulations behave differently, not that any particular formulation is a definitive measure of urban accessibility.

\subsection{Conclusion}

The divergence between cited closeness formulations and software implementations creates a reproducibility barrier in street network analysis. Mathematical \emph{Closeness} exhibits sign reversal; \emph{Normalised Closeness} yields negligible associations; \emph{Improved} and \emph{Harmonic Closeness} yield values that increase with subgraph size and proximity. Researchers can reduce ambiguity by specifying which formulation is used and documenting computational methods precisely. Open code and data are provided for reproducibility.
