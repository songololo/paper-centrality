Closeness centrality formulas cited in the street network analysis literature diverge from those implemented in computational packages, creating a barrier to reproducibility. A researcher implementing the commonly cited \emph{Normalised Closeness} formula will obtain different results from studies using domain-specific tools that implement variants such as \emph{Improved Closeness}. This distinction matters for localised (distance-threshold) analysis, now standard practice, because subgraph sizes vary by origin. Under these conditions, mathematical \emph{Closeness} exhibits sign reversal in associations with land uses and pedestrian activity, while \emph{Normalised Closeness} yields near-zero associations. By contrast, \emph{Improved} and \emph{Harmonic Closeness} preserve the intended interpretation, yielding values that increase with both node count and proximity: the former divides by \textbf{average} rather than \textbf{total} distance; the latter sums distance reciprocals, achieving similar scaling through a different mechanism.

We explain why this divergence occurs and assess the explanation empirically on the Madrid street network, using land-use and trip-count correlations as a diagnostic. The formulations differ in the direction and magnitude of their associations: \emph{Closeness} shows negative associations; \emph{Normalised Closeness} shows near-zero associations; \emph{Improved}, \emph{Harmonic}, and \emph{Gravity} formulations show positive associations. This pattern is consistent across metric and angular distances, length-weighted variants, and independent trip data. For localised street network analysis, we suggest using \emph{Improved Closeness}, \emph{Harmonic Closeness}, or similar formulations, and documenting the precise methods used. Open code and data are provided for reproducibility.
