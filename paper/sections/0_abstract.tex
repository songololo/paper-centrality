Street network analysis relates urban form to land-use intensity and travel behaviour, yet closeness centrality formulas cited in the literature diverge from those implemented in domain-specific software, creating a reproducibility barrier. This matters for localised (distance-threshold) analysis, now common practice, where reachable node counts vary by origin. \emph{Closeness} and \emph{Normalised Closeness} behave counter-intuitively across differently sized subgraphs; \emph{Improved} and \emph{Harmonic Closeness} scale correctly---the former dividing by \textbf{average} rather than \textbf{total} distance, the latter summing distance reciprocals.

We test these distinctions in Madrid using correlations with land-use intensity and trip counts as a diagnostic tool. The formulations diverge substantially: \emph{Closeness} shows negative associations ($\rho \approx -0.70$); \emph{Normalised Closeness} shows negligible associations ($\rho \approx -0.11$); \emph{Improved}, \emph{Harmonic}, and \emph{Gravity} formulations show positive associations ($\rho \approx +0.67$). This pattern persists across metric and angular distances, length-weighted variants, and trip data. For localised street network analysis, researchers should use formulations that scale appropriately with varying subgraph sizes and document the precise methods used. We provide open code and data.
