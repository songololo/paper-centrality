Closeness centrality formulas cited in the street network analysis literature diverge from those implemented in computational packages, creating a barrier to reproducibility. A researcher implementing the commonly cited \emph{Normalised Closeness} formula will obtain different results from studies using domain-specific tools that implement variants such as \emph{Improved Closeness}. This distinction matters for localised (distance-threshold) analysis, now standard practice, because subgraph sizes vary by origin. Under these conditions, mathematical \emph{Closeness} exhibits sign reversal, while \emph{Normalised Closeness} yields negligible associations. \emph{Improved} and \emph{Harmonic Closeness} yield values that increase with both node count and proximity: the former divides by \textbf{average} rather than \textbf{total} distance; the latter sums distance reciprocals, achieving similar scaling through a different mechanism.

We present a hypothesis for why this divergence occurs, which we then show holds empirically using correlations with land-use intensity and trip counts as a diagnostic. The formulations differ markedly: \emph{Closeness} shows negative associations; \emph{Normalised Closeness} shows near-zero associations; \emph{Improved}, \emph{Harmonic}, and \emph{Gravity} formulations show positive associations. This pattern is consistent across metric and angular distances, length-weighted variants, and independent trip data. For localised street network analysis, researchers should use \emph{Improved Closeness}, \emph{Harmonic Closeness}, or similar formulations and document the precise methods used. Open code and data are provided for reproducibility.
