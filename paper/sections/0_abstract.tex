Closeness centrality formulas cited in the street network analysis literature diverge from those implemented in computational packages, creating a reproducibility barrier. This distinction matters for localised (distance-threshold) analysis, now common practice, because subgraph sizes vary by origin. Under these conditions, mathematical \emph{Closeness} exhibits sign reversal in associations with land uses and pedestrian activity, while \emph{Normalised Closeness} yields weak, scale-dependent associations that reverse in sign across thresholds. By contrast, \emph{Improved} and \emph{Harmonic Closeness} yield values that increase with both node count and proximity: the former divides by \textbf{average} rather than \textbf{total} distance; the latter sums distance reciprocals. We explain why this divergence occurs and demonstrate it empirically on the Madrid street network using land-use and trip-count correlations as a diagnostic. For localised street network analysis, we suggest using \emph{Improved Closeness}, \emph{Harmonic Closeness}, or similar formulations, and documenting the precise methods used. Open code and data are provided.
