Closeness centrality formulas cited in street network analysis literature diverge from those implemented in domain-specific software, creating a reproducibility barrier for researchers implementing centrality from cited formulas or attempting replication without original code. This matters for localised analysis, now common practice, where reachable node counts vary by origin. In these cases, \emph{Closeness} exhibits sign reversal relative to its intended interpretation; \emph{Normalised Closeness} yields negligible associations; \emph{Improved Closeness} (dividing by average rather than total distance) and \emph{Harmonic Closeness} (summing distance reciprocals) behave as intended.

We present a hypothesis for why this divergence occurs, which we then show holds empirically using correlations with land-use intensity and trip counts as a diagnostic. The formulations differ markedly: \emph{Closeness} shows negative associations; \emph{Normalised Closeness} shows near-zero associations; \emph{Improved}, \emph{Harmonic}, and \emph{Gravity} formulations show positive associations. This pattern is consistent across metric and angular distances, length-weighted variants, and independent trip data. For localised street network analysis, researchers should use \emph{Improved Closeness}, \emph{Harmonic Closeness}, or similar formulations and document the precise methods used. Open code and data are provided for reproducibility.
